# design-exploration/

Visual design work-in-progress you're authoring yourself — separate from finalized handoffs.

Things that live here:

- Mockups you're iterating on (HTML, images, Figma exports)
- Design briefs you're drafting
- Visual comparisons — A/B/C layout options
- Notes about visual directions under consideration

When a direction is picked and bundled for implementation, it graduates to `../design-handoff/`.

For HTML prototypes generated for customer validation (Charlie Bailey's pattern), use `../../_pm/prototypes/` instead — those are about *capability* validation, not *visual* exploration.
