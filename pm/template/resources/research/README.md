# research/

External research, analysis, and reference material that informs project decisions.

Things that live here:

- Market overviews, competitive analyses
- Methodology references (project-management frameworks, scope-control techniques, requirement-engineering practices)
- Third-party reference docs worth preserving (vendor APIs, platform docs, compliance material)
- Analytics snapshots, traffic reports, user research outputs

Rule of thumb: if it's something you *read* to inform a decision, it goes here. If it's something you *write* to plan work, it goes in `../../_pm/TASKS.md`, the [skeleton](../../_pm/skeleton.md), or a milestone spec.
