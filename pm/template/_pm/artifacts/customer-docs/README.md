# Customer Docs

Documents the customer sent — briefs, requirements, contracts, existing system docs.

**Preserve original filenames** so you can refer to "the brief Sandy emailed" without renaming-induced confusion.

If the original is PDF or DOCX and you want Claude to read it directly, drop a `.md` version next to it (Claude can convert PDFs via the `pdf` skill / xlsx for spreadsheets). Keep both.

## What goes here

- Discovery briefs
- Existing system documentation provided by the customer
- Contracts and SOWs (if relevant to scope discussions — otherwise leave them in your contracts system)
- Previous developer's handoff docs, if you inherited a project
