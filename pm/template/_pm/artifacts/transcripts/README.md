# Transcripts

Meeting transcripts. Drop them here as they arrive.

**Filename:** `YYYY-MM-DD-<short-topic>.md` (e.g., `2026-06-04-discovery-call.md`).

**Source:** Zoom downloads, Granola exports, Otter, etc. Markdown or plain text. Preserve speaker labels if the tool produced them — Claude can use them to attribute requirements to specific stakeholders.

**Joe's pattern:** stop taking manual notes. Let the transcript carry the meeting. During the meeting, say "hey future me — this matters" out loud when something important comes up; that phrase becomes a searchable marker in the transcript.

## What to do with a new transcript

1. Drop it in this folder, dated.
2. Note it in today's `../../sessions/YYYY-MM-DD.md` entry — at least a one-line "had a discovery call with Sandy, transcript at \<path\>".
3. When ready to distill: ask Claude to generate user stories, persona updates, or acceptance criteria from it. Those go to `../../requirements/`.
4. Don't edit the transcript — it's the historical record. If the customer corrects something, note the correction in the parrot-back bundle, not in the transcript itself.
